CPU Allocation Info

CPU Period: Base time unit for CPU allocation (microseconds)

CPU Quota: Maximum CPU time the honeypot can use

Default setting (50%): Suitable for most honeypots

Increase values for CPU-intensive honeypots

What do these values mean?

Memory Limit: Maximum RAM the honeypot can use (in MB)

Memory Swap: Additional virtual memory when RAM is full

Setting Memory Swap equal to Memory Limit (default) effectively disables swap usage

Most honeypots run efficiently with 512MB RAM

Best Practices

1

Use realistic credentials

Common usernames like 'admin', 'root', and simple passwords will attract more attack attempts

2

Include default combinations

Add factory/default credentials for the service you're emulating (e.g., 'admin/admin' for routers)

3

Mix difficulty levels

Include a range from simple to medium-difficulty passwords to capture different attacker behaviors
